# מדריך סגנון — מסלולי "לימוד מ-0"

המטרה של כל מסלול: לקחת תלמיד ש**לא מכיר את הנושא בכלל** ולהביא אותו עד פתרון שאלת בגרות מלאה. נושא הזהב שלפיו כותבים הכל: [`math5/complex-numbers.ts`](./math5/complex-numbers.ts). כל נושא חדש מחקה את המבנה, הטון ורמת הפירוט שלו.

> כלל אצבע: אם משפט מניח ידע שלא הוסבר קודם במסלול — הוא נכשל. כל דבר נבנה מאפס.

## המבנה (8 חלקים — הסדר קבוע)

הטיפוס `LearningPath` (ב-[`types.ts`](./types.ts)) מחייב את כולם. שדה חסר = שגיאת קומפילציה.

| # | שדה | מה חייב להיות בו |
|---|---|---|
| 1 | `prerequisites` | מה צריך לדעת *לפני*. לכל פריט: רענון קצר (`refresher`) שמספיק כדי להמשיך, ובמידת הצורך `link` לנושא המלא ו-`selfCheck` (שאלת בדיקה אחת). |
| 2 | `intuition` | **לפני נוסחאות.** `hook` = הבעיה שהנושא פותר, בשפה אנושית. `body` = תמונה אינטואיטיבית/גרפית. `payoff` = "בסוף תדע ל…". מינימום סימבולים מפחידים. |
| 3 | `concepts` | הנושא מפורק ל**אטומים** קטנים. כל אטום: `plain` (הסבר פשוט, בלי נוסחה) ← `formal` (הגדרה/נוסחה **מנומקת**) ← `simplestExample` (הדוגמה הכי פשוטה) ← `whyItWorks`/`diagrams` (רשות). |
| 4 | `guidedExamples` | כל צעד = `action` + `why`. כל דוגמה פותחת ב-`methodChoice` ("איך יודעים באיזו שיטה לבחור"). מתחילים מ-`trivial` ועולים ל-`hard`. |
| 5 | `pitfalls` | לכל מלכודת: `mistake` (איך תלמיד טועה) + `correction` (למה זה שגוי ומה נכון) + `relatedConceptId` (קישור לאטום הרלוונטי). |
| 6 | `practice` | שלוש רמות: `level1` (מושג בודד), `level2` (שני מושגים), `level3` (בגרות מלאה רב-סעיפית). לכל שאלה/סעיף: `hints` הדרגתיים, פתרון מלא, ו-`reviewIfStuck` ("נתקעת? חזור ל…"). |
| 7 | `formulaSheet` | `quickReview` (תבליטי תזכורת) + `formulas` (עם משתנים מוסברים). לסקירה מהירה לפני מבחן. |
| 8 | `comprehensionCheck` | 5–6 שאלות MCQ קצרות שמאשרות שליטה לפני הנושא הבא. לכל אחת `explanation` של שורה. |

## עקרונות פדגוגיים

1. **אטומים, לא גושים.** עדיף 7 מושגים קטנים מ-3 גדולים. אם אטום דורש שני רעיונות נפרדים — פצל אותו.
2. **שום נוסחה "נזרקת".** ב-`formal` כל נוסחה מנומקת או מודגמת. אם אי-אפשר להוכיח בקצרה — לפחות תן `whyItWorks` עם אנלוגיה.
3. **תמיד "למה", לא רק "מה".** זו ההבחנה המרכזית בדוגמאות המודרכות. ה-`why` הוא ההיגיון שתלמיד מפנים, לא תיאור הפעולה.
4. **בחירת שיטה היא מיומנות.** `methodChoice` הוא הלב — תלמידים יודעים *לבצע* שיטה אבל לא יודעים *מתי* לבחור בה.
5. **רמזים נחשפים אחד-אחד.** מהעדין ("באיזה רבע הנקודה?") עד כמעט-התשובה. לא לחשוף הכל בבת אחת.
6. **כל `reviewIfStuck.conceptId` חייב להתאים ל-`ConceptAtom.id` קיים באותו מסלול.** כך הקישור גולל לאטום הנכון.

## כללי LaTeX ומתמטיקה

- **גרשיים כפולים תמיד.** בקובץ המקור כותבים `\\sqrt`, `\\dfrac`, `\\cos`, `\\text{cis}` — backslash כפול. (במחרוזות JS גרש בודד "נבלע".) זה תקף גם ב-template literals.
- **`$...$`** למתמטיקה בתוך משפט, **`$$...$$`** לנוסחה עצמאית (ממורכזת).
- **מספרים מרוכבים: `cis` ומעלות בלבד.** לעולם לא $e^{i\theta}$, לא רדיאנים, לא $\pi$. כותבים $45°, 90°, 180°, 360°$ — לא $\tfrac{\pi}{4}$. (סילבוס 5 יח"ל.)
- **טריגונומטריה: מעלות.** אותו כלל.
- בדוק כל תוצאה ידנית לפני שמירה. שגיאת סימן אחת שורפת את אמון התלמיד.

## עברית, RTL ו-bidi

- **לשון זכר** (בעל האפליקציה זיהה כבן).
- **בלי אימוג'י דקורטיבי בתוך התוכן.** הכותרות והרכיבים כבר נושאים אייקונים; אל תפזר 🎯⚠️📌 בתוך המשפטים.
- **bidi:** אל תסיים משפט עברי ב-`...$math$.` — הנקודה והמתמטיקה מתהפכות. במקום: שים את הנוסחה ב-`$$...$$` בשורה נפרדת, או הוסף מילה עברית אחרי המתמטיקה. בין שתי נוסחאות inline — חבר עם מילה עברית ("ו", "אז"), לא בפסיק.

## דיאגרמות

- השתמש ב-`DiagramSpec` מ-[`../lessons/types`](../lessons/types.ts).
- **רק `type: 'custom'` עם SVG גולמי** (או הפרימיטיבים הגאומטריים). **אסור `fn:` closures** (functionGraph עם פונקציה) — המסלול עובר גבול server→client, ופונקציה שם מפילה את העמוד (500) בפרודקשן. לעקומות, צייר SVG סטטי.
- ה-SVG הוא תוכן פנימי בלבד (בלי תג `<svg>` עוטף); `viewBox` ברירת מחדל `0 0 200 200`.
- כיתוב (`caption`) עובר דרך KaTeX — אפשר `$...$` בתוכו.

## הוספת נושא חדש

1. צור `math5/<topic>.ts` ובו `export const math5<Topic>: LearningPath = { … }`.
2. רשום אותו ב-[`index.ts`](./index.ts) במפתח `'math5:<שם הנושא בעברית>'` — **חייב להיות זהה** ל-`topic` של ה-`Lesson` הקיים (כדי שהקישורים יתאימו).
3. `npm run build` (בדיקת טיפוסים + smoke test) → commit → push.
4. הבאנר "מסלול לימוד מ-0" יופיע אוטומטית בדף הנושא הקיים (`/practice/...`) ברגע שיש מסלול.
